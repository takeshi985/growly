# Workbook and QR Foundation

A `Workbook` contains ordered `WorkbookPage` records. Each page has a unique,
URL-safe `qr_code_token` generated by the backend when none is supplied.

The browser fallback is `GET /qr/:token`. It shows the workbook page, optional
lesson, and safe links to the Growly demo. Tokens contain no child identity,
progress, or other private data.

Supported target types are `lesson`, `task`, `diagnostic`, and `course`.
Currently the browser fallback is informational. A future mobile application
can register the deep link `growly://workbook/:token`, resolve the token through
the backend, and open the matching lesson after parent/child context is known.

Actual QR image generation and print-ready PDF production are intentionally
deferred.
